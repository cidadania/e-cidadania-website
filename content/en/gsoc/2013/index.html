Introduction

These ideas are main tasks for the e-cidadania project. E-cidadania project is very young, it has only 1 year of development so it does not have an active community that supports it...yet. In this project the students will find a perfect test field for them, since they can contribute to the project without having any fear to be wrong, and also have the chance to participate in the early stage of a project.

Guidelines

Being an accepted student for the e-cidadania project is not as competitive as in KDE, but don't relax, you will have to adapt participative methodologies to an online (and therefore more limited) platform. We have the challenge to create a very versatile platform, and that is not easy.

There are some established guidelines when coding and documenting in e-cidadania. That rules are a must, you can see them in the project documentation

If for some reason you can't contact your mentor within 48 hours, you can try to contact the backup administrator, Armando Broz. Don't expect technical feedback from him.

Desirable prerequisites

The prerequisites of the e-cidadania platform are mostly the same for every project, they just vary in tiny little specs. Take in mind that even if we say "prerequisites", they are not decisive for selecting students. Students with no experience are welcome too.

The students are desirable to have:

  • Mid-level knowledge of the Python programming language
  • Some knowledge of the Django framework, previous knowledge is OK but it won't be decisive.
  • Good or very good knowledge of web technologies: CSS 2 and 3, HTML4, JavaScript, jQuery.

We repeat that this requisites are not decisive, after all every student must learn before anything, so don't be afraid to apply, your mentor is there to teach you :)

Core

Enhance AJAX security

Knowledge required: Django, AJAX, jQuery

The main goal is to strengthen the user permissions and access to the platform. The current security with files, spaces, and most of the user interactions are vaguely filtered and mostly with hardcoded instructions. The platform must use all the mechanisms available in the django framework to guarantee the roles and permissions effectiveness in the platform.

Graphical status of the process

Knowledge required: Django, jQuery

It would be nice to have some kind of system that allows the user to know graphically where and when h is on the process. Since every process can have multiple concurrent debates, proposals, or voting, the user can get easily lost. Some kind of timeline, at git/mercurial/bazaar, etc. style would be great. Every debate, proposal, etc. would be a branch, and they would be interconnected (meaning the relations between them) and at any time the user can view where he is, just looking at the "you are here" spot.

Create RESTful API

Knowledge required: Django, django-piston

Modern internet services should provide application interface for possible clients. Goal is to create API for e-cidadania project using django-piston framework. RESTful API allows to create statless, coherent client-server solution.

Modules

Automated reports

Knowledge required: Reporting tools, Django, Python

The automated reports system is one of the key features of e-cidadania. This tool must generate a PDF output on the space archive when finishing a process or on demand if the administrator requires it. There must be a predefined data model that will be automatically used after the process is closed, but the administrator must be able to specify the information needed to generate the report.

Enhance debates module

Knowledge required: Django, AJAX, jQuery

It would be great if debates system supported interaction between some other modules like the proposals module and allowed media content and comments in the notes system.

Make debates usable on mobile devices

Knowledge required: Responsive design, CSS, jQuery

Actual debate modules design allows it to be used almost in every place, but not in the current smartphones, where the "drag&drop" feature is gesture based. The idea is to develop a responsive design that when detects a smartphone (or a potential one) changes the design of the debate system allowing a better interaction to the user from their smartphone.

Testing

Create unit tests

Knowledge required: Django, unittest2

e-cidadania lacks of any unit test. The objective would be to create a complete test suite for the basic modules, so that guarantees its stability. The test would be realized with the test framework provided by django, using python's unittest.
